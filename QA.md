## Fix 2 — Currency Conversion

### Basic toggle
- [ ] Set all monetary inputs to 100,000 (MXN). Switch to USD → all three monetary fields convert to ~5,714 (100,000 / 17.5)
- [ ] Switch back to MXN → returns to 100,000 exactly (not 99,999.xx — the round-trip test)
- [ ] `rooms` field does NOT change when toggling currency (e.g., 1,500 stays 1,500)
- [ ] `occupied` field does NOT change when toggling currency
- [ ] `marketOcc` field does NOT change when toggling currency (e.g., 65 stays 65)

### Outputs update correctly
- [ ] RevPAR output updates when currency is toggled (same relative value, different unit)
- [ ] ADR output updates when currency is toggled
- [ ] GOPPAR output updates when currency is toggled
- [ ] Currency symbol/code in outputs matches the selected currency

### Precision — Round-trip conversion
- [ ] After converting MXN → USD → MXN on 100,000: result is **exactly** 100,000 (not 99,999.96)
- [ ] After converting MXN → USD → MXN on 17,500: result is **exactly** 17,500
- [ ] After converting MXN → USD → MXN on 780,000: result is **exactly** 780,000

### Unitless fields — NO conversion
- [ ] `rooms` field stays the same through both toggles (e.g., 1500 → 1500 → 1500)
- [ ] `occupied` field stays the same through both toggles (e.g., 975 → 975 → 975)
- [ ] `marketOcc` field stays the same through both toggles (e.g., 70 → 70 → 70)

## Fix 4 — GOPPAR Status (GOP Margin Threshold)

Now uses `gopMargin` percentage instead of hardcoded dollar amounts. Currency-agnostic and scale-agnostic.

**Test A — gopMargin = 35.9% → GREEN:**
- Input: rooms=1500, occupied=975, roomRevenue=780000, operatingCosts=500000
- gopMargin = (780000-500000)/780000 × 100 = 35.9%
- [ ] Status shows GREEN (good) ✓

**Test B — gopMargin = 20% → YELLOW:**
- Input: rooms=1500, occupied=975, roomRevenue=780000, operatingCosts=624000
- gopMargin = (780000-624000)/780000 × 100 = 20%
- [ ] Status shows YELLOW (warn) ✓

**Test C — gopMargin = 8% → RED:**
- Input: rooms=1500, occupied=975, roomRevenue=780000, operatingCosts=717600
- gopMargin = (780000-717600)/780000 × 100 = 8%
- [ ] Status shows RED (bad) ✓

**Thresholds:**
- [ ] gopMargin ≥ 30% → GREEN (good)
- [ ] gopMargin 15-29% → YELLOW (warn)
- [ ] gopMargin < 15% → RED (bad)

**Currency toggle test:**
- [ ] After toggling MXN → USD → MXN, GOPPAR status updates correctly (recalculates based on converted values)

## KPI Formula Verification

Use these known-correct values to verify the tool:
